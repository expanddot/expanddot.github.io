網站沒 SSL = 直接掉排名？手把手教你申請 SSL 憑證

SSL 憑證申請流程：

申請 SSL 憑證的步驟大致如下（適用於 DV、OV、EV 憑證）：

1️⃣ 選擇合適的 SSL 憑證類型（DV/OV/EV）→ 依據網站性質決定安全等級

2️⃣ 購買 SSL 憑證（選擇可信賴的憑證頒發機構，如 Sectigo、DigiCert、Certum）

3️⃣ 生成 CSR（Certificate Signing Request） → 由伺服器產生，包含網站資訊

4️⃣ 驗證身份（依據憑證類型，提供域名驗證或公司身份驗證文件）

5️⃣ 憑證核發 → CA 會在驗證成功後頒發 SSL 憑證

6️⃣ 安裝 SSL 憑證（在伺服器或主機後台設定 SSL）

7️⃣ 測試 SSL 是否正常運作（可用 SSL Labs 測試）

SSL 憑證申請的準備條件：

申請 SSL 憑證時，所需的資料會根據 憑證類型（DV、OV、EV） 而有所不同。以下是不同類型 SSL 憑證的申請準備條件：

1. DV（Domain Validation，域名驗證） 🔹 最簡單，最快速（幾分鐘內核發）

✅ 適合對象：個人網站、部落格、小型企業

✅ 驗證方式：只需要證明你擁有該網域

準備材料

1. 網域名稱（Domain Name）

  • 你必須擁有申請 SSL 憑證的網域，例如 example.com

2. 驗證方式（選擇一種）

  • Email 驗證 → 需要使用 `admin@example.com、webmaster@example.com` 等官方郵箱接收驗證信
  • DNS 記錄驗證 → 在 DNS 記錄中添加一條特定 TXT 記錄
  • HTTP 文件驗證 → 在網站根目錄放置特定驗證檔案（CA 會提供檔案內容）

3. CSR 檔案（Certificate Signing Request，部分機構需要）

  • 由伺服器生成，包含網站名稱、組織名稱等

2. OV（Organization Validation，組織驗證） 🔹 需要公司資料，通常 1-3 天核發

✅ 適合對象：企業、電子商務網站、政府機構

✅ 驗證方式：需要提供企業身份證明

準備材料

1. 網域名稱（Domain Name）

  • 與 DV 憑證相同

2. 公司/組織的官方資訊

  • 企業名稱（必須與政府註冊資訊一致）
  • 企業登記地址（符合當地政府商業登記的地址）
  • 聯絡電話（需要與公共資料匹配，例如黃頁或政府網站）
  • 企業註冊號碼或文件（例如工商營業執照、登記證明）

3. 驗證方式

  • 企業資料比對 → 憑證機構（CA）會檢查公司是否在官方商業登記庫（如 Dunn & Bradstreet, 政府機構網站）中
  • 電話驗證 → CA 會打電話給公司登記的官方號碼，確認負責人是否批准申請
  • Email/DNS/HTTP 驗證 → 與 DV 憑證相同

4. CSR 檔案（必須提供）

  • 由伺服器生成

3. EV（Extended Validation，擴展驗證） 🔹 最高等級驗證，通常 3-7 天核發

✅ 適合對象：大型企業、銀行、政府機構、金融科技公司

✅ 驗證方式：必須進行嚴格的公司身份驗證

準備材料

1. 網域名稱（Domain Name）

  • 與 DV/OV 憑證相同

2. 公司/組織的官方資訊（與 OV 相同，但需要更完整）

  • 企業法定名稱（與政府註冊一致）
  • 企業登記地址
  • 聯絡電話（需在公開的商業登記資料中可查）
  • 企業註冊文件（例如營業執照、公司註冊證書）
  • 政府機構提供的企業身份證明文件

3. 高層管理人員驗證

  • 申請人需要是 企業的法定代表人或高層管理人員（例如 CEO、IT 負責人）
  • CA 可能要求提供 公司授權書，以確認該人員可以申請 EV SSL
  • 電話回訪驗證：CA 會打電話給企業代表，確認申請細節

4. CSR 檔案（必須提供）

5. 額外的身份驗證

  • CA 可能會要求 律師或會計師簽署身份證明文件
  • 部分 CA 會使用 銀行對賬單或公用事業帳單 來驗證企業地址

申請不同 SSL 憑證所需時間

SSL 類型 申請所需資料 申請時間
DV SSL 只需驗證域名（Email/DNS/HTTP） 幾分鐘至數小時
OV SSL 企業登記資訊 + 網域驗證 1-3 天
EV SSL 企業完整驗證 + 高層管理人員確認 3-7 天
